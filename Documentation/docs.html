Getting Started

Little Babe is a small PHP framework for hybrid PWA development. Unlike other PHP frameworks with MVC (Model, View and Controller) pattern, Little Babe is designed for building hybrid progressive web applications (installable in cross-platform and multiple devices from mobile, tablets and desktop) with built-in functions for accessing localstorage browser API (for data persistency in the browser using Javascript), JSON file C.R.U.D (Create, Read, Update, Delete) operations and file uploads in PHP.

Install

Requirements:
You should have git installed on your machine.
If you're using Windows, open the terminal (command prompt) and navigate to htdocs folder and type:

git clone https://github.com/mjfrontendservices/little_babe

Or download the zip file.

Create New Page

Navigate to App/Pages/ and create a new php file, for example: about.php then copy the text from pageStructure.php and paste it on the new php file, in this case, on about.php. The pageStructure.php file content should be the template for all php files (pages) inside the App/Pages/, App/Post/ and App/Ajax/ folders.

* all get requests (web pages) should be stored in the App/Pages folder. This serves as your views folder.
* all post requests (form submission and backend processing) should be stored on App/Post folder. This serves as your controllers or logic folder, and
* all ajax requests will be stored on App/Ajax folder when you want to pass data to the server using Javascript.

Routing

In order to view your web pages in the browser, navigate to Routes/Routes.php file and register all your routes. For example: if you want to render the homepage. 

Route::set('home', function (){
    Generate::get('home');
});

The Route::set() method requires two arguments, the first argument is the route name. In this case, 'home'. And the second (callback function) argument function (){ Generate::get('home'); }) is the file name of the page you want to generate, in this case 'home.php', you can remove .php extension and make it 'home'. Now open your Chrome browser and type http://localhost/(project_folder)/home. The homepage should be rendered.

Other generate methods:
Generate::post('filename') - for post request (form submission, backend processing). This is linked to the App/Post/ folder.
Generate::ajax('filename') - for ajax request (sending data to the server using Javascript/JQuery). This is linked to the App/Ajax/ folder.

If you need to generate a webpage, use the Generate::get() method.
If you need to pass data to the server, use the Generate::post() method. And
If you need to pass data to the server using Ajax, then use the Generate::ajax() method.

Generate::install('installpage') - for custom install page (default)
Generate::docs('docs') - for the documentation page (default)

Assets

All your assets should be stored in the Assets folder.
Assets/css - for css files
Assets/files - for files (audio, video, pdf documents, etc...)
Assets/img - for images
Assets/js - for javascript files.
Assets/libs - for external libraries (default libraries: Minified Bootstrap Libraries, Font-Awesome, JQuery).
Assets/little_babe - for default files with Little Babe built-in core functions.

HTML Links and Scripts

HTML head tag is located in Includes/head.php. Link your css files, meta tags, icons, and other css libraries here.
Also HTML script tags are located in Includes/foot.php. Link your Javascript files and other javascript libraries here.

Database Connection

Navigate to Database/databaseCredentials.json configuration file for your database credentials. 

[
    {
        "server_name": "localhost",
        "server_username": "root",
        "server_password": "root",
        "database_name": "mydatabase"
    }
]

Write your own SQL scripts inside the Database/database.sql and import it on http://localhost/phpmyadmin.

Custom Install Page

Custom install page is a customized web page for installation promotion of your hybrid progressive web app. If the user installs your web app, it will function like a normal hybrid application on user's phone or computer. Feel free to edit, add or delete the Install/installpage.html file depends on the project requirements. Install page can be access through http://localhost/(project_folder)/appinstall route.

App Logo
Navigate to Install/logo folder to store the logo of your application. You should provide two different sizes of logo (96x96 and 192x192 pixels)

App Manifest
Application manifest file is located in Install/manifest.json. This is a configuration file of your PWA (how your app will look like on user's device, icons, app name, theme color, etc.) 

{
    "short_name": "App",
    "name": "My App",
    "description": "Lorem ipsum app manifest",
    "icons": [
        {
            "src": "./logo/logo-96x96.png",
            "type": "image/png",
            "sizes": "96x96"
        },
        {
            "src": "./logo/logo-192x192.png",
            "type": "image/png",
            "sizes": "192x192"
        }
    ],
    "theme_color": "#000",
    "start_url": "http://localhost/myapp/",
    "display": "standalone",
    "background_color": "#000",
    "orientation": "portrait"
}

short_name: Short name of your application.
name: The name of your application.
description: The detailed description of your application.
icons: Link of your application 96x96 and 192x192 logo.
theme_color: The theme color of your app.
start_url: When the user tap your app's icon, he/she will be redirected to this link (the homepage of your application).
You can add other properties on the manifest.json file for better application performance. Read more about other manifest properties.

JSON C.R.U.D Operations


Create, Read, Update, Delete

Read JSON file data

Reading data from the json file. 

$data = JSON::read('json_file.json')

print_r($data);

This will return a decoded associative array data. 

Array (
    [0] => Array (
        [id] => id60ffacf67e6b67.00721233
        [name] => Mark Jason Penote Espelita
        [age] => 21
        [address] => Brgy. San Isidro Northern Samar, Philippines 
    )
    [1] => Array (
        [id] => id40ccacf67e6b77.00776543
        [name] => Mark Penote Espelita
        [age] => 20
        [address] => Brgy. San Isidro Northern Samar, Philippines 
    )
)

Looping through data 

$data = JSON::read('json_file.json')

foreach($data as $key => $value){
    echo $value['name'] . '< br >';
}

Output: 

Mark Jason Penote Espelita
Mark Penote Espelita

Explain:
JSON::read() = JSON class has a read method for reading the data inside the json file. This requires one argument (json filename).

Insert Data to JSON file (push method)

Inserting data to json file (push method). 

JSON::push('json_file.json', array(
    'id' => JSON::id(),
    'name' => $name,
    'age' => $age,
    'address' => $address
));

// or you can do it this way ------

$obj = array(
    'id' => JSON::id(),
    'name' => $name,
    'age' => $age,
    'address' => $address
));

JSON::push('json_file.json', $obj);

Explain:
JSON::push() = JSON class has push method for inserting data to the bottom of json file. This requires two arguments (json filename and data you want to insert)
JSON::id() = simple method for generating random id.
'json_file.json' = json file where data will be inserted.
array ('key' => 'value') = associative array data to be inserted.
Note: If the json file doesn't exist. It will automatically create it for you.
Result:
json_file.json content should be like this. 

[
    {
        "id": "id60ffacf67e6b67.00721233",
        "name": "Mark Jason Penote Espelita",
        "age": 21,
        "address": "Brgy. San Isidro Northern Samar, Philippines"
    }
]

Insert Data to JSON file (unshift method)

Inserting data to json file (unshift method). 

JSON::unshift('json_file.json', array(
    'id' => JSON::id(),
    'name' => $name,
    'age' => $age,
    'address' => $address
));

// or you can do it this way ------

$obj = array(
    'id' => JSON::id(),
    'name' => $name,
    'age' => $age,
    'address' => $address
));

JSON::unshift('json_file.json', $obj);

Explain:
JSON::unshift() = JSON class has unshift method for inserting data on the top of json file. This requires two arguments (json filename and data you want to insert)
JSON::id() = simple method for generating random id.
'json_file.json' = json file where data will be inserted.
array ('key' => 'value') = associative array data to be inserted.
Note: If the json file doesn't exist. It will automatically create it for you.
Result:
json_file.json content should be like this. 

[
    {
        "id": "id60ffacf67e6b67.00721233",
        "name": "Mark Jason Penote Espelita",
        "age": 21,
        "address": "Brgy. San Isidro Northern Samar, Philippines"
    }
]

Remove data from JSON file

Removing one or more data from the JSON file using delete() method. 

JSON::delete('json_file.json', 'id', 'id60ffacf67e6b67.00721233');

Logic:
'Delete from json_file.json where id is id60ffacf67e6b67.00721233'
This will remove an object from a JSON file if the key (id) matches the value (id60ffacf67e6b67.00721233).
You can use other key instead of id. Example: name, age, address, status, etc...

Remove a user where name is Mark Jason Penote Espelita 

JSON::delete('json_file.json', 'name', 'Mark Jason Penote Espelita');

Remove a user where age is 21 

JSON::delete('json_file.json', 'age', 21);

Update data from JSON file

Updating one or more data from the JSON file using update() method. 

JSON::update('json_file.json', 'id', 'id60ffacf67e6b67.00721233', 'name', 'Marky Jason Penote Espelita');

Logic:
'Update the name to Marky Jason Penote Espelita from the json_file.json where id is id60ffacf67e6b67.00721233'
You can use other key instead of id. Example: name, age, address, status, etc...

Another example:
'Update the name to Marky Jason Penote Espelita from json_file.json where name is Mark Jason Penote Espelita' 

JSON::update('json_file.json', 'name', 'Mark Jason Penote Espelita', 'name', 'Marky Jason Penote Espelita')

Explain:
JSON::update() = JSON class has update method for updating data in the JSON file. This requires five arguments (json file, basis key, basis value, updatable key and new value)

File Upload

Upload any type of file you want in just one line of code. 

Files::upload($_FILES['file'], './Assets/img/', ['jpg', 'png', 'jpeg'])

Note:
There are limitations on file size when uploading a file.

Getting the filename of the uploaded file 

Files::upload($_FILES['file'], './Assets/img/', ['jpg', 'png', 'jpeg'])

$filename = Files::$filename;

echo $filename;

This will echo a filename of the uploaded file.

Explain:
JSON::upload() = JSON class has upload method for uploading a file to the server.
$_FILES['file'] = name of the file input from the form.
'./Assets/img/' = the path/folder where the file is stored after uploading, you can use another folder if you want.
['jpg', 'png', 'jpeg'] = valid extensions of the file to be uploaded. If the user upload a file with different file extension, it will return an error.

Local Storage Browser API


For data persistency in the browser.

Localstorage is a data storage in the browser. It can be utilize for storing temporary or permanent data in the client side, so the data will never lost during page reload.
Localstorage is found on developer tools/Application/Storage You can store a key - value pairs: objects, arrays, array of objects and more.
key = is the name of your data storage. In the image above, the key or the name of localstorage is Storage. You can name it whatever you want.
value = is the data being stored. In the image above, the value is an array of objects.

Read Local Storage Data

Reading data from localstorage. 

let data = localstorage_read('Storage')

console.log(data)

This will return a parsed javascript array of object. 

[
    {
        name: "Mark Jason",
        age: 21
    },
    {
        name: "Mark Jason",
        age: 21
    },
    {
        name: "Mark Jason",
        age: 21
    }
]

Looping through data.
Code in custom.js 

let data = localstorage_read('Storage');

data.foreach((element) => {
    console.log(element['name']);
    console.log(element['age']);
});

Output: 

Mark Jason
21
Mark Jason
21
Mark Jason
21

Insert Local Storage Data

Inserting data to localstorage. 

localstorage_insert('Storage', {
    name: "Marky Jason",
    age: 20
});

This will insert a new data to 'Storage' key.

Update Local Storage Data

Updating data to localstorage. 

localstorage_update('Storage', 'name', 'Marky Jason', 'name', 'Marky2 Jason')

Logic:
'Update name to Marky2 Jason in 'Storage' localstorage where name is Marky Jason'
This will update the name to Marky2 Jason on the 'Storage' key. Arguments:
1st. 'Storage' = name of localstorage (key).
2nd. 'name' = basis key.
3rd. 'Marky Jason' = basis value.
4th. 'name' = updatable key.
5th. 'Marky2 Jason' = new value.

Remove Data from Local Storage

Removing a string value from local storage.
String 

localstorage_delete('Storage', (del => del.name === 'Marky2 Jason'))

Removing an integer or float from string value from local storage.
Integer 

localstorage_delete('Storage', (del => del.age === parseInt('21')))

Logic:
If we will execute the String:
'Remove an object where name is Marky2 Jason from 'Storage' localstorage key.'
Output: The Marky2 Jason has been removed.
